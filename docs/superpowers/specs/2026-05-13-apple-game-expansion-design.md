# 사과게임 기능 확장 및 UI 개편 설계서 (2026-05-13)

## 1. 개요
현재의 단일 모드 선택 방식에서 탈피하여, '메인 화면 -> 상세 설정 -> 게임 실행'으로 이어지는 사용자 경험(UX)을 제공하고, 다양한 게임 플레이 옵션을 추가함.

## 2. 주요 변경 사항

### 2.1 UI 구조 개편
- **메인 화면 (Main Scene)**: '혼자하기', '같이하기 (Coming Soon)' 버튼 배치.
- **설정 화면 (Settings Scene)**: '혼자하기' 클릭 시 진입. 상세 옵션 선택 후 '게임 시작' 버튼으로 게임 진입.
- **게임 화면 (Game Scene)**: 상단 HUD에 현재 선택된 모드 정보(예: 17x10 | 10의 배수) 표시 추가.

### 2.2 신규 게임 옵션
- **시간 모드**: 120초 제한 시간 또는 무한 모드 선택.
- **배열(그리드)**: 가로/세로 크기 사용자 정의 (최소 5x5 ~ 최대 50x50).
- **클리어 가능 여부**: 
    - 랜덤: 기존 Rejection Sampling 방식.
    - 퍼펙트 시드: 100% 클리어 가능 판 생성 (사용자 정의 크기 대응).
- **클리어 타입**:
    - 오리지널: 드래그 영역 합계가 정확히 10인 경우 제거.
    - 10의 배수: 드래그 영역 합계가 10, 20, 30 등 10으로 나누어 떨어지는 경우 제거.

## 3. 기술 설계

### 3.1 GameEngine (core/engine.js)
- `evaluateSelection(r1, c1, r2, c2, config)`: `config.clearType`에 따라 `sum === 10` 또는 `sum % 10 === 0 && sum > 0` 조건을 체크하도록 수정.

### 3.2 SeedGenerator (core/generator.js)
- `perfectGen(rows, cols)`: 기존 고정 크기 가정을 제거하고, 전달받은 `rows`, `cols`에 맞춰 영역 분할 및 합계 배분 로직 수행.

### 3.3 App Controller (web/app.js)
- `config` 객체를 통해 게임 상태 관리.
- 화면 전환 함수 `showScene(sceneId)` 구현.
- 게임 종료 시 메인 화면이 아닌 설정 화면 혹은 메인 화면으로 선택적 복귀 기능.

## 4. 검증 계획 (TDD)

- **Generator Test**: 5x5, 50x50 등 극단적인 크기에서 퍼펙트 시드가 오류 없이 생성되는지 확인.
