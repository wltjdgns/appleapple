# 사과 게임 UI/UX 개선 설계서 (2026-05-14)

## 1. 개요
사과 게임의 전반적인 사용자 경험(UX)과 시각적 디자인(UI)을 개선하여 가독성을 높이고 기능을 확장한다.

## 2. 화면별 변경 사항

### 2.1 메인 화면 (MainScreen)
- **문구 삭제:**
    - `Orchard · Sum to Ten` 삭제
    - 좌측 하단 버전 정보 (`v0.1 · integrated orchard build...`) 삭제
    - 우측 하단 푸터 테이프 (`토토 농장 직송`, `★ ★ ★ ★ ★`, `Hand-Picked Daily`) 삭제
- **버튼 추가:**
    - `나의 바구니`, `리더보드` 버튼 하단에 `[버그제보]` 및 `[문의/개선점]` 버튼을 가로로 나란히 배치.
    - 각 버튼 클릭 시 피드백 입력 모달 노출.
- **크레딧 추가:**
    - 화면 최하단 중앙에 `Made By Sunghoon Ji` 문구를 작고 회색 톤(`--ink-mute`)으로 추가.

### 2.2 설정 화면 (SettingsScreen)
- **가독성 향상:** 전체적인 텍스트 크기 상향 조정.
- **옵션 명칭 변경:**
    - 시간 모드: `2분 (제한 시간)` -> `시간내로`, `무한 시간` -> `시간제한없이`
    - 클리어 가능 여부: `랜덤 생성 (일반)` -> `랜덤`, `퍼펙트 시드 (100% 가능)` -> `가능`
    - 클리어 타입: `오리지널 (합계 10)` -> `오리지널`, `10의 배수 (10, 20...)` -> `10의 배수`
    - 라벨 변경: `테마 색상 (사과 색상)` -> `사과 색상`
- **레이아웃 변경:**
    - 상단의 `뒤로 가기` 버튼과 우측 하단의 `게임 시작` 버튼을 제거.
    - 모든 설정 옵션 최하단(사과 색상 선택지 아래)에 `게임 시작`과 `뒤로 가기` 버튼을 가로로 나란히 배치.

### 2.3 게임 화면 (GameScreen)
- **정보 배치 최적화:**
    - 사이드바에 있던 `PROGRESS` 바와 `가이드` 텍스트를 사과 배치 판 위쪽, 타임 바 아래쪽의 가로 영역으로 이동.
    - 가로로 얇고 긴 형태로 배치하여 시선 분산을 최소화.
- **조작 버튼 개선:**
    - `[색상 변경]` 버튼 크기 확대 및 강조.
    - `[그만하기]` 버튼 위치 변경: 사과 배치 판 하단 중앙에 `[그만하기]`와 `[나가기]` 버튼을 나란히 배치.

### 2.4 결과 화면 (ResultScreen)
- **종료 메시지 세분화:**
    - 정상 종료 외에 "더 이상 가능한 수가 없습니다" 등의 상태별 메시지 표시.
- **버튼 레이아웃 최적화:**
    - `[같은 모드로 한번더]` 버튼의 가로 길이를 적절히 조절.
    - 그 아래에 `[다른 설정]`, `[메인으로]`, `[리더보드]` 버튼을 균등한 간격으로 나란히 배치.

## 3. 신규 기능 구현: 피드백 모달

### 3.1 기능 명세
- `MainScreen` 내부에 `FeedbackModal` 컴포넌트 구현.
- 제목(`title`)과 내용(`content`) 입력 필드 제공.
- `전송` 버튼 클릭 시 Firebase Firestore의 `feedback` 컬렉션에 데이터 저장.
- 저장 데이터 구조: `{ type: 'bug' | 'suggestion', title, content, timestamp, uid }`

### 3.2 UI 디자인
- 배경: 반투명 다크 그레이 오버레이.
- 컨테이너: 따뜻한 종이 질감(`--paper-warm`), 둥근 모서리, 그림자 효과.

## 4. 기술 스택 및 제약 사항
- **Framework:** React
- **Styling:** CSS-in-JS (Inline styles) 및 전역 CSS 변수
- **Database:** Firebase Firestore
- **제약:** 외부 라이브러리 추가 없이 기존 프로젝트 구조 내에서 구현.
